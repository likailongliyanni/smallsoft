# SmallSoft AI 后台

这个仓库是“一套 AI 收费后台 + 多个独立小软件”的基础工程。

当前第一个产品模块：

## EXCEL自动化

页面入口：

- `/excel-automation`

当前功能：

- 图片提取
- 表格整理：上传多个原始表（可多 sheet）和可选模板表，AI 规划合并方式——「堆叠合并」（同义字段归类后上下拼接，如「商品名称/品名/名称」归为一组）或「按键匹配」（join，如订单表+快递表按电话对上，把快递单号接到订单行）。浏览器本地计算每列值形态/唯一值数和跨表键值交集作为证据（只上报统计，不上报列值），AI 据此选匹配键——用户说的键在数据里不存在时按证据自动纠正，并在计划里说明实际用了什么键；确认页展示一句话合并计划 + 匹配预检（多少行能对上），支持人工改合并方式/主表/匹配键，确认后浏览器本地合并下载 xlsx；`/api/excel-automation/table-merge/plan` 只接收表头、样例和统计摘要；引擎单测：`node tests/table-merge-local.test.cjs`
- 数据清洗（通用脏 Excel 智能结构化）：上传任意乱表 + 口语化要求，浏览器本地识别多段表头/数据区域（表头可不在首行、可跨多行）、按结构信号删合计/小计/重复表头/页脚等噪声行、「同上/〃」回填、日期金额标准化，输出干净表 + 异常待确认（带置信度和来源行号）+ 被删行记录 + 字段映射 + 处理日志，多 sheet 一并打进下载的 xlsx。**不写死任何业务字段**：目标字段由表头+列形态+用户要求动态推断；登录后 AI 规划字段（`/api/excel-automation/table-tidy/plan` 只接收表头候选、列形态统计和少量样例），未登录走本地规则。引擎单测：`node tests/table-tidy-local.test.cjs`；UI 纯函数单测：`node tests/table-tidy-ui.test.cjs`；浏览器测试台：`/tidy-harness.html`

旧入口：

- `/spreadsheet-images` 已移除

能力边界：

- 浏览器本地读取 `.xlsx`，文件和图片不上传服务器
- 浏览器本地预览多个 sheet、单元格和内嵌图片
- 登录后调用 AI 规划规则，但只发送表格摘要、样例行、图片锚点和自然语言要求
- AI 的职责是理解用户不规范、口语化的描述，并把它转换成精确可执行的字段映射、命名模板、分文件夹规则和图片处理规则
- 浏览器本地提取 xlsx 内嵌图片
- 根据自然语言生成命名和分文件夹规则
- 支持按 sheet / 货号 / 颜色 / 品牌 / 分类等字段命名
- 支持一行多图自动追加图片序号
- 支持裁剪白边、统一尺寸、轻微锐化
- 浏览器本地生成图片 ZIP，并在 ZIP 内附带 `manifest.csv` 和 `plan.json`

运行依赖：

- 前端依赖新版 Chrome / Edge 的 `DecompressionStream` 用于本地解压 xlsx
- 预览、裁剪、缩放、锐化、打包均在浏览器完成
- 后端 PHP 用于账号、额度、管理后台和 AI 规则生成，不接收用户 xlsx 原文件或图片
- `/api/excel-automation/image-extract/plan` 只接收轻量摘要，用于让大模型处理不规则表格规则

额度规则：

- 当前本地 MVP 不上传文件；AI 规则生成需要用户登录
- 后续商业版可以记录“摘要 + 规则 + 图片数量”做扣费，不上传 xlsx 和图片本体
