### EXP-033 禁止输出 json 语言标记

**问题背景**：
有些模型会把 JSON 放进 Markdown 代码块，或在内容前单独输出一行 `json`：

```text
json
{
  "version": "1.0",
  "actions": []
}
```

这不是合法的 JSON DSL。解析器会把第一个非空白字符读成 `j`，不是 `{`，从而报错“AI 返回内容不是有效的 JSON DSL”。

---

## 铁律

最终回复的第一个非空白字符必须是 `{`。
最终回复的最后一个非空白字符必须是 `}`。

---

## 禁止输出

- 不要输出单独的 `json`。
- 不要输出 ````json`。
- 不要输出 ```。
- 不要输出 `JSON:`、`以下是 JSON`、`AI 返回 JSON` 等前言。
- 不要输出任何 Markdown 代码块语言标记。

---

## 正确输出

```text
{
  "version": "1.0",
  "name": "录制流程",
  "actions": []
}
```

注意：上面的示例仅说明内容形态。真实最终回复不要带 ```text，也不要带任何代码块。

---

## 最后自检

发送前检查原始文本：
- 去掉首尾空白后，第一字符是不是 `{`？
- 去掉首尾空白后，最后字符是不是 `}`？
- 开头是否残留 `json`、```json 或 ```？

如果开头不是 `{`，必须删除前缀后再输出。
