# 智能截图软件 · Electron 界面版 — 开发进度

> 在原版 `desktop/snap-saver`（tkinter）基础上，用 Electron 重写界面，
> 后端复用原版 `snap_saver.py` 的全部成熟逻辑（截图钩子 / AI / 排版 / 服务器对接）。
> **原版完整保留，不动**，出问题时对照查。

## 架构

```
snap-saver-electron/
├── snap_saver.py        原版后端逻辑（复用，含 CtrlDragHook 全局钩子）
├── backend.py           命令分发器：Electron ↔ Python（行式 JSON 双向 + 事件推送）
└── electron/
    ├── main.js          主进程：开窗口、启动 Python 子进程、IPC、截图覆盖窗
    ├── preload.js       主界面安全桥
    ├── overlay-preload.js / overlay.html   截图选框可视化覆盖层
    ├── capture.html / capture.js           （旧版全屏框选，已被 overlay 取代）
    ├── index.html / style.css / renderer.js  界面 + 逻辑
    └── node_modules/    依赖（Electron 运行时 + Quill），不入库
```

通信协议（每行一个 JSON）：
- 命令：`{id, cmd, args}` → 回应 `{id, ok, data/error}`
- 后台事件（截图进度等）：`{event, ...}` → 主进程转发为 `py-event`

## 本轮修的 bug

- ✅ 图文配对：批量导入时「图→说明」严格交替，不再图堆一起、说明全挤底下
  （`insertDocImage(item, atEnd=true)` 强制追加文末）
- ✅ AI 修复「图片不存在/失败」：根因是 `list_folder_images`/`list_shots` 返回**相对路径**，
  backend 工作目录对不上就找不到文件。改为返回 `p.resolve()` 绝对路径。
- ✅ 截图任务完成后停止钩子（`_advance` 全部截完调 `self.stop()`）
- ✅ 选框「缓慢缩小」：根因是 `screen.screenToDipPoint` 对移动点转换不稳定逐帧漂移。
  改为固定缩放因子线性换算（物理坐标 / scaleFactor）。
- ✅ 文档编辑器最大化按钮（切换 `.doc-maximized` 铺满全窗）

