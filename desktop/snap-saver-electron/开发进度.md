# 智能截图软件 · Electron 界面版 — 开发进度

> 在原版 `desktop/snap-saver`（tkinter）基础上，用 Electron 重写界面，
> 后端复用原版 `snap_saver.py` 的全部成熟逻辑（截图钩子 / AI / 排版 / 服务器对接）。
> **原版完整保留，不动**，出问题时对照查。

## 架构

```
snap-saver-electron/
├── snap_saver.py        原版后端逻辑（复用，含 CtrlDragHook 全局钩子）
├── backend.py           命令分发器：Electron ↔ Python（行式 JSON 双向 + 事件推送）
└── electron/
    ├── main.js          主进程：开窗口、启动 Python 子进程、IPC、截图覆盖窗
    ├── preload.js       主界面安全桥
    ├── overlay-preload.js / overlay.html   截图选框可视化覆盖层
    ├── capture.html / capture.js           （旧版全屏框选，已被 overlay 取代）
    ├── index.html / style.css / renderer.js  界面 + 逻辑
    └── node_modules/    依赖（Electron 运行时 + Quill），不入库
```

通信协议（每行一个 JSON）：
- 命令：`{id, cmd, args}` → 回应 `{id, ok, data/error}`
- 后台事件（截图进度等）：`{event, ...}` → 主进程转发为 `py-event`

## 编辑器深度打磨（2026-06-22 续 · 代码审查后）

> 用户反馈「编辑器还不够完美」，逐块代码审查后定位真问题，一刀一刀修、每刀实测。

- ✅ **第一刀·治内存（图多不卡）**：以前每张图把整张原图读成 base64 dataURL 塞进 DOM，
  十几张大图就卡死。改为后端 `make_thumb` 生成 **800px 缩略图**显示（小、快），
  原图路径单独记在 `docImagePaths`；**导出前 `read_full_image` 换回原图高清**，
  编辑流畅 + 成品清晰两全。后端新增 `make_thumb` / `read_full_image` 命令。
- ✅ **第二刀·图片勾选框替代飘逸拖拽手柄**：删掉整套 `docImageResizer`（body 浮层靠
  `getBoundingClientRect` 算坐标会飘）。改为每张图左上角 **`✓ 导出 / ✕ 不导出` 文字徽章**
  （做在包裹图片的 `<p>::before` 上，纯 CSS 跟图走绝不飘）；点徽章区切换、点图中间仅选中。
  **去掉选中态大绿框**（用户嫌「大框包裹整图」），选中只给徽章描白边。
  判定区放大到覆盖整个徽章，解决「点击没反应」。
- ✅ **第三刀·导出成品清洗**：导出时在 HTML 副本上 ①删被标「✕ 不导出」的图及其说明

  ③缩略图换原图高清。`docClear` 改真正空白靠 placeholder 提示；标题兜底 `使用说明`→`详情页`。
- 🧪 验证：`verify-click`（真实尺寸图+真实点击坐标：徽章命中切换/中间不误触/选中无大框/徽章渲染）

## 新功能·AI 商品主视觉（电商场景重构）（2026-06-23）

> 用户澄清：要的不是 PIL 裁剪拼接/宫格/海报，而是 **多张参考图 → 视觉分析锁定商品 →
> AI 重新生成一张全新的、像同一次拍摄出来的专业电商图**（参考用户给的 GPT 黑咖啡成片）。
> 在现有「AI 修复」同一条阿里云 multimodal-generation 框架内改造，PIL「组合图」按钮被替换。

- ✅ **服务器 `SceneReconstructService`**（两步）：
  ①`analyze()` 视觉模型(qwen3.6-plus)读全部图 → 结构化 JSON
  (product_identity / image_roles / scene_plan / avoid)；
  ②`render()` 把 JSON+用户选项拼最终提示词 → 多图 content `[{image},{image},…,{text}]`
  调万相多模态端点 → 出新图。提示词强约束：锁定商品数量/朝向/品牌logo/包装文字，
  不换商品、不编卖点文字、不拼接、不宫格、不加水印价格。
- ✅ **接口 `/desktop/scene/reconstruct`**：多图上传 + 6 选项，扣 1 张生成额度，
  返回 PNG。错误友好化（Key/权限/欠费/限流/审核）。模型后台可换
  （`ModelConfig(purpose=scene_reconstruct)` 或 `config/ai.php`，默认复用 wan2.7-image）。
- ✅ **桌面 `cmd_reconstruct_scene`**：多张参考图按勾选顺序压到 1024px → `images[]`
  multipart 上传 → 存「AI主视觉」目录。（本地 HTTP server 实测：端点/鉴权/多图/字段/落盘全对）
- ✅ **界面「AI 商品主视觉」弹窗**：AI 修复里勾选 2-6 张参考图（第 1 张=主体）→ 弹窗设
  比例/用途/风格/还原强度/留文案位/补充要求 → 生成（分阶段文案反馈，约 30-90 秒）→
  预览+打开成品。错误处理齐全。（隐藏窗口实测：选图→开弹窗→6 选项有序传参→结果 UI 更新 全过）
- ⚠️ **未验证**：真实阿里云出图质量。需服务器部署该接口后用真实商品图测；
  万一 wan2.7-image 多图融合不够好，后台换更强的多图生成/编辑模型即可，代码不动。
- 🗑️ PIL `compose_images` 后端命令已无前端调用（保留为无成本离线兜底，不影响主流程）。
