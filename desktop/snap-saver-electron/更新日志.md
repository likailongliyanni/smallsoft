# 智能截图软件 + 服务器 更新日志

## 2026-06-15

这次同时改了**桌面软件**（智能截图软件 / snap-saver）和**服务器**（Laravel 后台，部署在 tools.haobanfa.online）。

### 一、桌面软件（snap-saver/snap_saver.py）

1. **「保留原图」开关**（工具栏，AI 智能修复 旁）
   - 默认不勾：修复结果**直接覆盖原图**，不再生成备份文件夹。
   - 勾选：修复前把原图备份到同目录 `_含水印原图`，新旧分开。
   - 选项持久化，下次打开沿用。

2. **「结果核对」对比窗 + 二次修复**
   - 修复完成后自动弹出滚动窗，每张图 `原图 → 修改后` 并排展示，不用再进文件夹一张张翻。
   - 双击「修改后」缩略图看大图。
   - 对效果不满意的勾选「需要二次修复」，点「二次修复选中」在当前结果上再处理一次（每张再扣 1 张额度）；二次修复后重新弹窗，原图始终对照最初的原图。
   - 实现要点：因为默认直接覆盖原图，修复前会把原图缓存到临时目录供对比，临时目录在对比窗关闭后清理。

3. **截图「冻结整屏」**（省 AI 钱的关键）
   - 按下 Ctrl+左键拖动的瞬间，整屏冻结成一张静态图铺在最上层。
   - 拖框时鼠标落在冻结图上，网页收不到鼠标移动 → 不再弹悬停放大镜/红框等浮层 → 不会被截进商品图（省去后续 AI 去除的花费）。
   - 所见即所得，框到什么就存什么；多显示器下选区超出主屏时自动回退实时抓屏。

4. 使用说明同步更新（含新流程、套餐价目、冻结屏说明）。

> ⚠️ 桌面端这些改动要**重新打包 exe** 才对外生效：`pyinstaller 智能截图软件.spec`

### 二、服务器（Laravel）

1. **图片修复模型切到 `wan2.7-image`**（修复 access_denied）
   - 原因：账号对 `qwen-image-2.0` 是「不支持开启」，一直 access_denied；`wan2.7-image` 可用。
   - wan2.7-image 与旧模型用**同一个** multimodal-generation 同步端点、同样的图+文结构，切换≈改模型名；但万相**不支持** `negative_prompt` / `prompt_extend`，已从请求里删掉（防改角度/防合并的约束保留在正向 prompt 里）。

2. **图片修复模型「后台可配置」**（百炼上新模型，后台改名即可，不改代码/不重新部署）
   - 复用现有 `model_configs` 表（purpose 字段），新增 `image_repair` / `image_detect` 两类。
   - 后台「模型配置 →🖼️ 图片修复模型」可改模型名；保存后**优先级最高**（盖过 .env / 配置文件）。
   - 读取顺序：后台配置 → config → 兜底常量（wan2.7-image）。

3. **新软件编号免费额度 50 → 10 张**（只影响新注册，老用户不变）。

4. **充值套餐**（单一来源在 `config/platform.php`）
   - 9.9 元 / 20 张，20 元 / 50 张，30 元 / 80 张，50 元 / 150 张，100 元 / 400 张。
   - 后台「调整额度」面板按套餐渲染快捷按钮，点一下自动填张数+备注。

### 三、部署备忘（宝塔 + PHP 8.2）

服务器根目录 `/www/wwwroot/tools.haobanfa.online`：

1. 覆盖上传上面 6 个服务器文件。
2. **宝塔重载 PHP 8.2**（清 OPcache，否则加载不到新 .php）。
3. 清缓存（**用 config:clear，不要 config:cache**——管理员登录用 `env()` 直接读，cache 后会读不到）：
   ```bash
   cd /www/wwwroot/tools.haobanfa.online
   /www/server/php/82/bin/php artisan config:clear
   /www/server/php/82/bin/php artisan route:clear
   /www/server/php/82/bin/php artisan view:clear
   /www/server/php/82/bin/php artisan cache:clear
   ```
4. 数据库不用迁移（沿用已有 `model_configs` 表）。
5. 后台「模型配置 →🖼️ 图片修复模型」填 `wan2.7-image` 保存。

> 管理员账号密码存在服务器 `.env` 的 `ADMIN_USERNAME` / `ADMIN_PASSWORD`（明文）。
