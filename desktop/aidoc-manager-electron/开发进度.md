# AI 档案管理 EXE 开发进度

## 产品约定

- 用户无需注册账号。
- 软件自动读取本机序列号，档案软件后缀为 `AIDOC`。
- 付费和额度仍在 `tools.haobanfa` 后台管理。
- 本地密码只用于打开本机资料库，后台不保存用户本地密码。
- 原始文件不上传。
- 识别按页计费，一页不管多少字都算 1 页。
- 新设备默认赠送 50 页。
- 进度条必须按页展示，避免用户误以为软件卡死。

## 第一版已落地

- 新建项目：`desktop/aidoc-manager-electron`
- Electron 主进程：`electron/main.js`
- 安全桥：`electron/preload.js`
- Vue 前端：`src/App.vue`
- 本地后端：`backend.py`
- 本地配置目录：`%APPDATA%/HaobanfaAIDoc`
- 本地资料库：用户选择目录下的 `aidoc.db`

## 已实现能力

1. AIDOC 序列号

   原始格式：

   ```text
   001122AABBCC-AIDOC
   ```

   界面显示：

   ```text
   00-11-22-AA-BB-CC-AIDOC
   ```

2. 本地密码

   - PBKDF2-SHA256
   - salt 随机生成
   - 只保存 hash
   - 解锁状态只存在当前进程，不写入配置文件

3. 资料库目录

   - 用户选择本地资料库位置
   - 自动创建 `files/` 和 `aidoc.db`
   - 支持复制管理和仅建索引

4. 扫描导入

   支持扩展名：

   ```text
   pdf, jpg, jpeg, png, webp, bmp, tif, tiff, docx, xlsx, xlsm, txt, csv
   ```

5. 页数统计

   - PDF：优先 `pypdf`，失败后正则兜底
   - 图片：1 张 = 1 页
   - Word：按文字量估算
   - Excel：按 sheet 数
   - 文本：按字符/行数估算

6. 进度事件

   Python 每处理一个文件都会推送：

   ```json
   {
     "event": "aidoc_progress",
     "stage": "count",
     "stage_label": "正在统计页数",
     "current_file": "营业执照.pdf",
     "done_pages": 12,
     "total_pages": 80,
     "success_pages": 12,
     "skipped_pages": 0,
     "failed_pages": 0
   }
   ```

## 2026-06-26 新增：证件识别引擎（核心价值已落地一半）

新增 `docintel.py`（识别引擎）+ backend.py 的 `analyze_document` 命令。已用「知识库训练样本」里的真实证件验证通过。

- **取证据双路**：用 PyMuPDF(fitz) 抠文字层；扫描件/图片渲染成页图 PNG。10 份真实样本里 5 份有文字层、5 份扫描件，路由判断正确（`needs_vision` 阈值 30 字）。**不依赖 poppler，打包 EXE 干净。**（requirements.txt 已加 PyMuPDF）
- **13 种证件类型 + 专属字段 schema + 识别提示词**：移植自网页参考版 `DocumentIntelligenceService`。
- **规则层**（免费、可离线兜底）：类型关键词判断、日期/证件号正则、按标签抠公司名。真实样本上已免费抠出 `A2240045682201003C` / `润本生物技术股份有限公司` 等。
- **AI 主力识别**：`recognize_via_server` 把文字或页图(base64)发服务器端点 `/api/desktop/document/recognize`（按页计费）；离线/端点未就绪时自动退回规则。
- **归一化 + 合并**：AI 为主、规则补缺；日期统一 YYYY-MM-DD；schema 外字段过滤；置信清洗。
- **落库**：DB 扩 17 个识别列（document_type/各字段/extra_fields/review_status/duplicate_of_id…），结果进 `pending_review`。`connect_db` 自动迁移老库。
- **查重**：扫描导入时按 sha256 挡完全相同（已有）；识别后按**同证件号**标 `duplicate_reason='certificate'`。
