# 图片整理器 · 项目改造说明

> 日期：2026-06-11
> 项目位置：`E:\0521\picture_organizer`
> 主程序：`app.py`（单文件，约 1700 行，Tkinter + Playwright + PIL）
> 打包：PyInstaller 单文件 exe（`图片整理器.spec`），当前 66.7 MB

## 一、这个软件是干什么的

电商商品图片采集整理工具。

使用者准备一份「A列：商品名称、B列：商品链接」的表格（Excel/CSV/TXT，或直接复制粘贴），
软件逐行打开商品网页，把采集到的图片自动按下面的结构保存：

```
输出目录\
└── 商品名称（A列）\
    ├── 主图\
    │   ├── pic1.jpg
    │   └── pic2.jpg
    └── 详情\
        ├── pic1.jpg
        ├── pic2.jpg
        └── pic3.jpg
```

## 二、旧版怎么工作的（现状）

旧版的取图由**软件**完成，有两种方式：

1. **点击原图**：内置一个 Playwright Chromium 浏览器，向网页注入 JS 脚本，
   按住 Ctrl 点击网页图片时，脚本自动分析出原图地址（srcset、懒加载属性、背景图等），
   软件直接下载原图并转成 JPG 保存。
2. **拖拽截图**：全屏遮罩上按住 Ctrl 拖拽框选屏幕区域，截屏保存。

保存逻辑：先存主图、再存详情，**数量够了自动跳到下一行**并自动打开下一条链接。

## 三、为什么要改

- 截图/自动识图的**效果不好**：识别到的图不一定是真正的原图，截图质量也不行。
- 决定改成：**人工在浏览器里找到真正的原图自己保存**，软件只负责「打开网页」和「图片归档」。
- 推进方式改成**手动点「下一行」**，不再数量够了就自动跳，节奏由人控制。

## 四、新版怎么工作（目标）

软件定位从「取图工具」变成 **「自动开网页 + 自动归档改名」工具**。

### 工作流程

1. 导入 A列名称 + B列链接（不变）
2. 按 F9 开启工作模式：主窗口隐藏，置顶小浮窗常驻，用**系统默认浏览器**打开当前行链接
3. 人工在网页里找到真正的原图，右键「图片另存为」存到**浏览器下载目录**
   （软件默认监控系统「下载」文件夹，监控路径可在软件里修改）
4. 软件监控到新图片，**自动搬走并改名**：
   `输出目录\商品名\主图\pic1.jpg`（统一转 JPG、透明背景补白、质量可调、pic1/pic2…连续编号）
5. 主图存完，浮窗上点「**详情**」切换 → 之后保存的图归入 `详情\`
6. 点「**下一行**」→ 切到下一行、自动打开下一条链接、类别自动重置回「主图」
7. 重复，直到列表做完

### 关键规则

| 项目 | 规则 |
|---|---|
| 主图/详情区分 | 浮窗手动切换按钮，点「下一行」后自动回到「主图」 |
| 主图数量/详情数量设置 | **只做进度提示**（如「主图 3/4」），不触发任何自动跳转 |
| 下一行防手滑 | 当前商品 0 张图时点「下一行」会弹确认提醒 |
| 图片格式 | 不管原格式（webp/png/…）统一转 JPG |
| 命名 | 前缀可设（默认 pic）+ 序号，每个类别独立编号 |

### 浮窗（工作模式的控制中心）

显示：当前商品名、主图/详情进度、当前归档类别
按钮：**主图/详情切换**、**下一行**、跳过当前、复制上一个、**撤销上一张**、显示程序、结束

> 撤销上一张：万一下载目录混进不相干的图被误归档，一键删除并退回编号。

### 下载目录监控的边界处理

- 只认**开启工作模式之后**新出现的文件，目录里原有文件一律不动
- 忽略浏览器下载中的临时文件（`.crdownload` / `.tmp` / `.part`），等下载完成再搬
- 只处理图片格式（jpg / jpeg / png / webp / gif / bmp / avif），其他文件不碰
- 连续快速保存多张时，按文件出现顺序依次编号

## 五、保留与删除清单

### 保留

- 文件导入（xlsx/csv/txt，多编码兼容）、剪贴板导入、表头自动识别
- 列表 Treeview（进度、状态展示）
- 跳过当前、复制上一个（SKU 规格不同但图相同的场景）
- F9 全局热键（显示/隐藏主窗口）
- 文件夹名非法字符自动替换、输出目录结构

### 删除

- 「Ctrl+点击取图」整套：Playwright 内置浏览器、JS 注入脚本、取图键逻辑（约 600+ 行）
- 「拖拽截图」整套：全屏遮罩框选、ImageGrab 截屏
- 取图键设置（Ctrl/Alt/Shift/空格/F8）、保存方式切换

### 删除带来的收益

- 打包体积从 66.7 MB 大幅缩小（Playwright 是大头）
- 启动速度明显变快
- 不再依赖内置浏览器的登录状态，用自己日常的浏览器，登录、插件都是现成的
